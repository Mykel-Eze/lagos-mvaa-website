Lagos MVAA API

Developer reference for the Motor Vehicle Administration Agency portal API — with a live console to test every endpoint from your browser.

Getting started #

Every request is made against a single Base URL. The default points at the test environment:

https://licensetest.permit.org.ng/api/v1

You can change it in the Base URL field at the top of the page — it’s saved in your browser and used by every “Try it” console below.

Three steps to test a protected endpoint:
1. Open Sign in and send your credentials.
2. Click “✓ Use as session token” on the response to capture the session.
3. Every 🔒 Session endpoint will now send that token automatically.

Authentication #

Sign-in does not return plain tokens. Instead it returns a single server-encrypted session envelope in the data field:

{
  "status": 200,
  "message": "login successful",
  "success": "true",
  "data": "{\"iv\":\"…\",\"txt\":\"…\",\"tag\":\"…\",\"used\":0,\"purpose\":\"login\"}"
}

Treat that data string as an opaque session token. Do not decrypt it on the client — store it verbatim and send it back on every authenticated request via the request header:

X-Portal-Session-Id: <the data envelope>
Endpoints marked 🔒 Session require this header. The console adds it automatically from the Session Token field above. Endpoints marked Public need no token.
Note: the envelope appears to be single-use (used:0 / purpose). A token may stop working after the backend redeems it — re-run sign-in to get a fresh one.

Conventions & errors #

All requests and responses use application/json. Successful responses generally wrap their payload in a data field. Validation and other failures return a consistent error shape:

{
  "success": "false",
  "error": "Bad Request Exception",
  "timestamp": "2026-06-10T05:02:53.554Z",
  "path": "/api/v1/portal/accounts/update-company-account/…",
  "status": 400,
  "details": [
    "address.flatNumber must be longer than or equal to 2 characters"
  ]
}
StatusMeaning
200 / 201Success.
400Validation error — see the details array for the exact fields.
401 / 403Missing, invalid, or expired session — sign in again.
404Resource not found.
500Server error.

Testing & CORS #

The “Try it” consoles send real fetch requests from your browser to the Base URL. Because the API lives on a different origin, the server must allow this page’s origin via CORS.

If a request fails with a network/CORS error but the API is up, the server hasn’t whitelisted your origin. Options: open this page from an allowed origin (e.g. the deployed app domain), have the backend allow it, or copy the request into a REST client (Postman/Insomnia) or the generated cURL-style call.

Nothing is sent anywhere except the Base URL you set. Your session token is stored only in this browser’s localStorage.
